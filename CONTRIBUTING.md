# Contributing Guide

Thank you for contributing to Context Driven Prompt.

This repository is built for practical, high-quality prompts that others can reuse.

## Contribution Workflow

1. Fork the repository.
2. Create a feature branch.
3. Add your prompt file in the correct category.
4. Verify formatting and clarity.
5. Open a pull request.

## Where To Add Prompts

Use this path format:

prompts/<domain>/<topic>/<prompt-name>.md

Examples:

- prompts/education/discrete-mathematics/proof-assistant.md
- prompts/coding/python/debugging-runbook.md
- prompts/content/seo/blog-outline-generator.md

## Prompt File Checklist

Before opening a pull request, ensure your prompt includes:

1. Title and short description.
2. When to use the prompt.
3. The full copy-ready prompt block.
4. Input variables or assumptions.
5. Expected output format.
6. One simple example.

## Quality Expectations

1. Be explicit and unambiguous.
2. Avoid vague instructions.
3. Prefer process-driven prompts over short-answer prompts.
4. Keep language professional and easy to follow.
5. Remove unnecessary repetition.

## Pull Request Checklist

1. Prompt path follows repository structure.
2. Filename is descriptive and lowercase with hyphens.
3. Prompt is tested with at least one model.
4. Example output is realistic.
5. Markdown renders correctly.

## What Not To Submit

1. Harmful or unsafe content.
2. Copyrighted material without rights.
3. Prompts with personal or sensitive data.
4. Low-context prompts with no output requirements.

## Questions

If you are unsure where a prompt belongs, open an issue with:

1. Prompt purpose
2. Target users
3. Proposed folder path
